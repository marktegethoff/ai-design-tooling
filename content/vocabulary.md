# Vocabulary

**Eyebrow:** Pillar 02

**Subtitle:** The words designers use to describe Claude Code determine how they behave with it. Vocabulary is not soft — it governs whether the system compounds or stagnates.

---

## The critical reframes

| Old framing | New framing | Why it matters |
|---|---|---|
| Training program | Infrastructure installation | Training implies receiving. Infrastructure implies building something that compounds. |
| Config file | Behavioral contract | Config files get ignored. Contracts get debated, updated, and owned. |
| AI tool | AI collaborator | Tools are passive. Collaborators have context, preferences, and a working relationship. |
| Prompt | Brief | Designers already know how to write briefs. This reuse of existing vocabulary reduces resistance. |
| Getting Claude to do X | Directing the collaborator | Agency stays with the designer. The AI executes under direction, not autonomously. |
| AI-generated output | AI-assisted output | Authorship remains with the designer. This is not a semantic nicety — it affects IP ownership framing. |

---

> **Introduce vocabulary before tools.**
> Run the vocabulary workshop before the first Claude Code session. Designers who arrive at the terminal already knowing what a behavioral contract is will configure their CLAUDE.md differently — more deliberately — than designers who encounter the file without that frame.
